# Aprende Spanish Breadth Plan

**Status:** Planning doc for review.
**Date:** 2026-05-31
**Scope:** Post-MVP breadth expansion toward real Spanish fluency, with vocabulary breadth as the primary axis.
**Source of truth:** `SPEC.md` and the existing `PLAN.md` content-vetting model. If this roadmap conflicts with those files, update the spec first.

## Goal

Move Aprende from a narrow offline MVP toward a broad Spanish-learning app by expanding the amount, coverage, and usefulness of bundled Spanish content without weakening the existing client-only, source-checked, review-gated design.

Vocabulary breadth is the spine of this roadmap. Grammar, phrases, exercises, review modes, and fluency practice should be added only where they help learners acquire, retain, and use more high-value words.

## Non-Goals And Guardrails

- Do not change shipped MVP behavior just to support this plan. New content should be additive unless a phase explicitly calls for a migration.
- Do not add runtime-generated learning content. Sentences, translations, accepted answers, glosses, and grammar explanations still enter through the build-time content pipeline.
- Do not ship unreviewed or sourceless learning content. The `REVIEWED` publish gate remains mandatory.
- Do not add server-backed features, accounts, sync, or leaderboards. This roadmap stays compatible with the client-only offline product.
- Do not start with broad UI redesign. Breadth comes from content coverage, metrics, curriculum structure, and review surfaces first.

## Prioritization Principles

1. **Vocabulary coverage first.** Each phase should increase the learner's useful Spanish word coverage or improve the system's ability to choose the next words.
2. **Coverage before volume.** Add metrics and source decisions before adding thousands of rows, so growth can be measured and reviewed.
3. **Sentences make words usable.** A lexeme is not considered learner-ready until it has enough vetted sentence contexts and accepted answers.
4. **Grammar supports lexical use.** Grammar expansion should unlock useful word families and phrase patterns, not become a separate abstract syllabus.
5. **Review gates are product features.** Content correctness is part of the learner experience, not just build hygiene.

## Breadth Targets

These targets are planning bands, not hard release promises. They are ordered by instructional value and review cost.

| Band | Primary learner outcome | Approximate vocabulary target | Notes |
|---|---:|---:|---|
| A1 Core | daily survival, basic descriptions, high-frequency verbs | 500 reviewed lemmas | First real expansion target after metrics/source decisions. |
| A2 Working | travel, errands, personal history, routines, opinions | 1,200 reviewed lemmas | Requires more sentence variety and tense coverage. |
| B1 Bridge | narration, common media, explanations, plans, needs/wants | 2,000 reviewed lemmas | Matches the original v1 ambition in `SPEC.md`. |
| Fluency Breadth | wider topics, idioms, collocations, register | 3,000+ reviewed lemmas/phrases | Deferred until the 2,000-lemma spine is reliable. |

A lexeme counts toward these targets only when it has source, license, review status, gloss, frequency rank or inclusion reason, and enough vetted exercises to teach and review it.

## Phase 0 - Coverage Baseline And Source Decision

**Priority:** highest
**Produces:** a measurable content expansion baseline and a final source policy for vocabulary growth.
**Depends on:** existing content pipeline and `content_manifest.json` shape.
**Ships independently:** yes; no learner-visible behavior change required.

Work:

- Add a content coverage report generated by `tools/content-pipeline/`.
- Report counts by lexeme, POS, CEFR band, frequency bucket, source, license, `vettingStatus`, sentence coverage, exercise coverage, and accepted-answer coverage.
- Define "learner-ready lexeme" in the report:
  - reviewed lexeme row
  - reviewed primary gloss
  - reviewed source/license
  - at least 2 reviewed sentence contexts for nouns/adjectives/adverbs
  - at least 4 reviewed sentence contexts for high-value verbs
  - at least 1 production exercise and 1 recognition/review exercise
- Resolve the frequency-list decision currently tracked as `PLAN.md` O1:
  - choose the canonical frequency source for ordering
  - document license obligations
  - document whether ShareAlike data is segregated from authored content
  - record attribution requirements used by the in-app credits screen
- Create a tracked coverage snapshot file, for example `tools/content-pipeline/coverage_snapshot.json`, generated but reviewable.

Acceptance criteria:

- Running the content pipeline produces a coverage report and manifest without changing `content.db` behavior.
- The report clearly distinguishes raw lexemes from learner-ready lexemes.
- The report fails or exits non-zero when content-bearing rows are sourceless, unlicensed, or not `REVIEWED`.
- The frequency source decision is documented with license and attribution consequences.
- A worker can identify the top missing A1/A2 vocabulary gaps from the report.

Deferred:

- Bulk vocabulary ingestion.
- New lesson UI.
- Grammar expansion.
- Audio/speech content.

## Phase 1 - A1 Core Vocabulary Spine

**Priority:** highest after Phase 0
**Produces:** the first broad, reviewed vocabulary expansion pack centered on the most useful A1 words.
**Depends on:** Phase 0 source decision and coverage report.
**Ships independently:** yes; as a content-only app update if schema does not change.

Work:

- Build the first target list of roughly 500 learner-ready lemmas from the chosen frequency source plus manual A1 curation.
- Prioritize:
  - function words and question words
  - core verbs such as `ser`, `estar`, `tener`, `ir`, `hacer`, `querer`, `poder`, `decir`, `saber`, `venir`
  - daily nouns: people, home, food, time, places, transport, body, weather
  - high-utility adjectives and adverbs
  - classroom/app vocabulary only where it appears in real exercises
- For every included lexeme, add vetted sentence coverage and accepted answers.
- Expand Path nodes only enough to teach and review this word set:
  - Basics
  - People
  - Food and drink
  - Home
  - Places
  - Time and routines
  - Wants and needs
  - Questions
- Keep grammar limited to what the word set needs: present tense, articles/gender, basic adjective agreement, negation, subject pronouns, basic `ser`/`estar`.
- Add or update content tests that assert the A1 pack is internally connected: no orphan lexemes, no lesson with unreviewed content, no target item without SRS identity.

Acceptance criteria:

- At least 500 learner-ready lexemes are reported as `REVIEWED`.
- At least 90% of included A1 lexemes have two or more reviewed sentence contexts; high-value verbs have four or more.
- Every new exercise targets a specific `srs_item` identity and has at least one accepted answer.
- The Path can introduce the A1 pack in a coherent order without requiring later-phase grammar.
- Existing MVP flows still pass without behavior changes.

Deferred:

- Preterite/imperfect contrast.
- Subjunctive.
- Conversation mode.
- Idioms and regional variants beyond common accepted-answer variants.
- Large-scale audio.

## Phase 2 - Vocabulary Coverage Metrics In Product And Tooling

**Priority:** high
**Produces:** actionable vocabulary coverage tracking for maintainers and lightweight learner-facing progress signals.
**Depends on:** Phase 0 metrics; Phase 1 enough content to make metrics meaningful.
**Ships independently:** yes.

Work:

- Add build-time curriculum metrics:
  - learner-ready lexemes by CEFR band
  - reviewed sentence contexts per lexeme
  - exercise type distribution per lexeme
  - weak spots: high-rank words with too few contexts
  - overrepresented words and duplicated prompts
- Add testable content budgets:
  - minimum context count per learner-ready lexeme
  - maximum unintroduced target words per node
  - no content pack with `UNVETTED` rows
  - no node whose exercises all use the same exercise type
- Add learner-facing progress summaries only if they can reuse existing surfaces:
  - Words screen filters by topic, CEFR band, strength, and seen/unseen
  - Home screen "words learned" count backed by SRS seen state
- Avoid promising exact fluency percentages in UI. Internal coverage estimates can stay in reports.

Acceptance criteria:

- CI or the local pipeline can fail a content pack for coverage regressions.
- Maintainers can answer "what are the next 100 missing high-value words?" from the report.
- Learners can inspect known and weak vocabulary without changing the core lesson loop.
- Metrics use stored content/SRS data, not runtime inference from generated text.

Deferred:

- Personalized adaptive curricula.
- User-selected goals by profession or hobby.
- Cross-user analytics.

## Phase 3 - A2 Expansion: Tenses, Phrases, And Daily Situations

**Priority:** medium-high
**Produces:** the next broad vocabulary band, with grammar and phrase coverage that make the words usable in real situations.
**Depends on:** Phase 1 A1 spine; Phase 2 content budgets.
**Ships independently:** yes, likely as a content plus domain-logic update.

Work:

- Expand to roughly 1,200 learner-ready lemmas.
- Add topic clusters:
  - travel and directions
  - shopping and money
  - health and body
  - work and school
  - family and relationships
  - hobbies and preferences
  - chores and errands
  - common public signs and instructions
- Add phrase/collocation entries as first-class reviewable items where single-lemma learning is insufficient:
  - `tener que`
  - `ir a + infinitive`
  - `me gusta`
  - `hay`
  - `por favor`, `de nada`, `lo siento`
  - common preposition pairs and time expressions
- Expand grammar only where needed for the A2 vocabulary:
  - preterite
  - imperfect
  - near future
  - present perfect
  - object pronouns
  - informal commands
  - common comparatives
- Add vetted contrast sets for grammar that cannot be safely generated, especially `ser`/`estar`, preterite/imperfect, `por`/`para`, and object pronoun order.

Acceptance criteria:

- At least 1,200 learner-ready lexemes/phrases are reported as `REVIEWED`.
- Each new grammar capability has vetted examples, accepted answers, and tests for grading/conjugation behavior.
- Phrase items have stable SRS identities separate from incidental lexeme matches.
- A2 nodes do not depend on B1-only grammar.
- Coverage report identifies remaining high-value gaps after the A2 pack.

Deferred:

- Present subjunctive except fixed expressions if individually reviewed.
- Open-ended free writing.
- Speech recognition requirements.
- Domain-specific vocabulary packs.

## Phase 4 - B1 Bridge Vocabulary And Productive Fluency

**Priority:** medium
**Produces:** a broad mid-level curriculum that lets learners narrate, explain, and understand more natural Spanish.
**Depends on:** Phase 3 A2 expansion.
**Ships independently:** yes, but likely needs both content and exercise expansion.

Work:

- Expand to roughly 2,000 learner-ready lemmas/phrases.
- Add B1-oriented topic clusters:
  - opinions and reasons
  - plans and obligations
  - problems and solutions
  - news and media basics
  - feelings and social situations
  - housing and services
  - common abstract nouns
  - connectors and discourse markers
- Add grammar that unlocks B1 expression:
  - conditional
  - present subjunctive in high-frequency frames
  - relative clauses
  - pluperfect/compound tense recognition where useful
- Add exercise support for productive fluency without runtime content generation:
  - sentence transformation from vetted source pairs
  - fill-in-the-blank for pronouns, tense, connectors, and prepositions
  - phrase assembly for collocations
  - mixed review sessions by weak topic or weak grammar tag
- Extend content tagging to include topic, grammar point, register, and phrase pattern where needed.

Acceptance criteria:

- At least 2,000 learner-ready lexemes/phrases are reported as `REVIEWED`.
- Each B1 grammar tag has reviewed explanation text or is omitted from learner-facing teaching.
- Mixed review can select by weak lexeme, phrase, or grammar tag without using unreviewed generated prompts.
- B1 content remains fully offline and attribution-compliant.

Deferred:

- Full CEFR certification alignment.
- Open conversation/roleplay.
- LLM tutor features.
- Multi-language support.

## Phase 5 - Fluency Breadth Packs

**Priority:** later
**Produces:** optional breadth packs beyond the core 2,000-word spine.
**Depends on:** Phase 4 and stable content operations.
**Ships independently:** yes, preferably as separately gated content packs if app size requires it.

Work:

- Add curated packs for:
  - restaurant and cooking
  - travel depth
  - workplace Spanish
  - school and parenting
  - medical basics
  - media and internet
  - regional vocabulary notes
- Add collocations, idioms, and false friends only with strong source and review coverage.
- Consider Play Asset Delivery if content size threatens the base install.
- Keep each pack independently measurable and removable from the roadmap if review cost is too high.

Acceptance criteria:

- Each pack has a declared target audience, source list, vocabulary target, review owner, and attribution output.
- A pack cannot ship unless its learner-ready count and sentence coverage meet its own budget.
- The base app remains usable offline without downloading optional later packs after install.

Deferred:

- Server-hosted pack catalog.
- User-generated content.
- Dynamic AI-authored content.
- Paid subscriptions or purchases.

## Content Model Additions To Consider

These are not Phase 0 requirements, but later phases may need them.

- `phrase` or generalized `learning_item` table for collocations and fixed expressions.
- `topicTag` and `grammarTag` tables for coverage metrics and targeted review.
- `content_pack` metadata if vocabulary expands beyond one bundled course asset.
- Reviewer notes or review batch IDs for auditability.
- Difficulty calibration fields for phrase items, not just lexemes.

Any schema addition should be justified by a phase acceptance criterion, not added speculatively.

## Deferred Items

- Runtime AI tutor, generated sentences, generated corrections, or generated grammar explanations.
- Cross-device sync, accounts, cloud progress, or social competition.
- Full speech-first curriculum.
- Stories, roleplay, video, or open conversation modes.
- Other languages or Spanish-for-non-English-speakers courses.
- CEFR certification claims.
- Monetization, subscriptions, billing, or gated content purchases.
- Large UI redesign unrelated to vocabulary breadth.

## Highest-Value Next Implementation Step

Implement **Phase 0: Coverage Baseline And Source Decision**, starting with the content coverage report.

Suggested first worker task:

1. Extend `tools/content-pipeline/build_content_db.py` to emit a coverage report next to `content_manifest.json`.
2. Include counts for lexemes, learner-ready lexemes, POS, CEFR band, frequency buckets, sentence contexts per lexeme, exercise targets, sources, licenses, and vetting statuses.
3. Add a pipeline test or command mode that fails when a supposedly learner-ready lexeme lacks reviewed sentence/exercise coverage.
4. Document the chosen frequency source and license policy in the pipeline README.

This step is highest value because it does not alter MVP behavior, makes vocabulary breadth measurable, resolves the source-selection blocker before bulk authoring, and gives future workers a concrete list of the next words to add.
